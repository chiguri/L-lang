*言語Lのインタープリタ

計算モデル論入門という本（http://www.amazon.co.jp/%E8%A8%88%E7%AE%97%E3%83%A2%E3%83%87%E3%83%AB%E8%AB%96%E5%85%A5%E9%96%80%E2%80%95%E3%83%81%E3%83%A5%E3%83%BC%E3%83%AA%E3%83%B3%E3%82%B0%E6%A9%9F%E6%A2%B0%E3%81%8B%E3%82%89%E3%83%A9%E3%83%A0%E3%83%80%E8%A8%88%E7%AE%97%E3%81%B8-Information-Science-Engineering-F5/dp/4781911358）の4章で使われていた疑似言語のインタープリタのSchemeによる実装。
R5RS準拠ならば動くはず。

一応実行前に簡単な構文検査と型検査を行う。
構文検査は引数の数（手続きは引数の数を確かめないので注意）、型検査は手続きと変数の区別を付けている。

いくつか本とは異なる実装にしている。主に以下の点。
-(f# X 10)などで呼び出している部分は(sharp f X 10)のように関数をsharpとして呼び出すことにしている。（#は取扱が面倒なため）
-手続き呼び出しにパラメータを与えるとXからではなくX1から始まるようにしている。（本の手法が不統一であるため）
-callマクロを導入している。（引数付き定義及び呼び出し）
-処理中の環境が出力されるdisplay関数を導入している。（デバッグ用）
-(execute)で標準入力から開始、(execute-file "filename")でファイルから読み込んで開始。（その他にも種々の実行方法があるが、コードから読むこと）
